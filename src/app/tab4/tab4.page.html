Privacy & Security Requirements
Mobile Inventory Management System

This page explains the main privacy and security requirements considered in this inventory management system.

It allows users to view, add, edit and delete inventory records, so it must handle data carefully and reduce the risk of accidental or unauthorised changes.

1. Data Collection Only collect inventory-related information

The app only collects the information required for inventory management, such as item name, item ID, category, featured status, supplier, price, quantity, stock status and optional notes.

It does not ask users to enter unnecessary personal information or store sensitive personal data.

2. Secure Communication Protect data during network transfer

The system communicates with a remote REST API. All requests should be sent through HTTPS so that data is protected while travelling between the mobile device and the server.

It applies to all main operations, including loading inventory items, adding new items, editing existing items and deleting records.

3. Input Validation Reduce invalid or unsafe data

The app validates user input before submitting data to the server. Required fields must not be empty, while optional fields such as notes can be left blank.

Numeric fields such as price and quantity should only accept valid integer values. Enum fields, including category, featured status and stock status, should use controlled selections instead of unrestricted text input.

4. Data Integrity Prevent accidental updates and deletions

The search function uses exact item name matching so that users do not accidentally edit or delete the wrong record. This is especially important when item names are similar.

Delete operations require an additional confirmation message before the request is sent. This helps prevent accidental data loss.

5. User Responsibility Safe use on mobile devices

Users should protect their mobile devices with a passcode, fingerprint or face unlock. They should avoid using the application on unsafe public networks when managing important inventory data.

Users should also check the item details carefully before editing or deleting a record.

6. Server and Database Security Protect backend resources

The server and database could restrict direct access to inventory data. In a production system, authentication and authorisation should be used so that only approved users can create, update or delete records.

Error messages do not expose technical server details to users. The system could show clear messages when a request fails.

Summary Privacy and security design goal

The goal of system requirements is to keep inventory data accurate, reduce accidental changes, protect data during transfer and encourage safe mobile usage.

These requirements support a more reliable and responsible inventory management application.
